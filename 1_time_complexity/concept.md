### 🔹 시간 복잡도란?

알고리즘에서 시간 복잡도는 주어진 문제를 해결하기 위한 연산 횟수이다.
일반적으로 1억번의 연산에 대한 수행 시간은 1초로 간주한다.

### ⏱ 시간 복잡도 유형

| 표기법 | 이름 | 의미 |
| --- | --- | --- |
| Ω(n) | 빅-오메가 | 최선일 때의 연산 횟수 |
| θ(n) | 빅-세타 | 일반적인 경우의 연산 횟수 |
| O(n) | 빅-오 | 최악일 때의 연산 횟수 |

#### 💡 코딩 테스트에서 사용하는 시간 복잡도 유형은?

> 모든 테스트 케이스의 수행에 대해 합격해야 하므로 **빅-오 (O(n))** 를 사용한다.

### 🧮 정렬 알고리즘별 시간 복잡도

| 정렬 | 설명 | 최선 Ω | 평균 θ | 최악 O | 공간 복잡도 | 안정 정렬 |
| --- | --- | --- | --- | --- | --- | --- |
| 버블 정렬 (Bubble Sort) | 인접한 두 원소를 비교해 크기 순으로 자리를 교환 | O(n) | O(n²) | O(n²) | O(1) | ✅ |
| 선택 정렬 (Selection Sort) | 매 회전마다 최솟값을 찾아 앞쪽과 교환 | O(n²) | O(n²) | O(n²) | O(1) | ❌ |
| 삽입 정렬 (Insertion Sort) | 정렬된 부분에 새 원소를 알맞은 위치에 삽입 | O(n) | O(n²) | O(n²) | O(1) | ✅ |
| 병합 정렬 (Merge Sort) | 배열을 반으로 나눠 각각 정렬 후 병합 | O(n log n) | O(n log n) | O(n log n) | O(n) | ✅ |
| 퀵 정렬 (Quick Sort) | 피벗을 기준으로 작은/큰 값을 나눠 재귀적으로 정렬 | O(n log n) | O(n log n) | O(n²) | O(log n) | ❌ |
| 힙 정렬 (Heap Sort) | 힙(완전 이진 트리) 구조를 이용해 최댓값/최솟값을 반복 추출 | O(n log n) | O(n log n) | O(n log n) | O(1) | ❌ |

> ⚠️ 퀵 정렬은 평균적으로 가장 빠르지만, 이미 정렬된 배열 등 피벗이 한쪽으로 치우치는 경우 최악의 경우 O(n²)까지 나빠질 수 있다.

#### 📝 예제 — 버블 정렬 / 삽입 정렬

```python
# 버블 정렬: 인접한 두 원소를 비교하며 자리를 교환 (최악/평균 O(n²))
def bubble_sort(arr):
    n = len(arr)
    for i in range(n - 1):
        for j in range(n - 1 - i):
            if arr[j] > arr[j + 1]:
                arr[j], arr[j + 1] = arr[j + 1], arr[j]
    return arr

# 삽입 정렬: 앞에서부터 정렬된 부분에 값을 끼워 넣음 (거의 정렬된 데이터에서 O(n)에 가까움)
def insertion_sort(arr):
    for i in range(1, len(arr)):
        key = arr[i]
        j = i - 1
        while j >= 0 and arr[j] > key:
            arr[j + 1] = arr[j]
            j -= 1
        arr[j + 1] = key
    return arr
```

#### ✅ 요약

- 시간 복잡도 = 문제 해결에 필요한 연산 횟수 (1억 연산 ≈ 1초)
- Ω(최선) / θ(평균) / O(최악) 3가지 표기법 존재
- 코딩 테스트에서는 모든 케이스를 통과해야 하므로 **O(n) 기준**으로 판단
- 버블/선택/삽입 정렬은 구현이 단순하지만 평균 O(n²)로 느림
- 병합/퀵/힙 정렬은 O(n log n)으로 대용량 데이터에 적합 (단, 퀵 정렬은 최악의 경우 O(n²))
