# 공부 내용 정리

사용자가 설명한 알고리즘/개념 내용을 해당 섹션 디렉토리의 `concept.md`에 정리합니다.

## 절차

1. 대상 section 디렉토리를 확인한다.
   - 사용자가 디렉토리를 지정하지 않으면 대화 맥락(방금 학습한 주제, 열려있는 파일 등)으로 추정하고, 애매하면 사용자에게 확인한다.
   - 디렉토리가 없으면 `{순번}_{주제}` 형식(기존 폴더 명명 규칙)으로 새로 만든다.
2. 기존 `concept.md`가 있으면 Read로 먼저 읽어 스타일과 기존 내용을 파악한다.
3. 아래 스타일 규칙에 따라 내용을 작성/보강한다.
4. 저장소 루트의 `README.md` 목차 표에 새 섹션이 없으면 한 줄 추가한다.
5. 정리하면서 추가하면 좋을 만한 내용(예: 예제 코드, 시간/공간 복잡도 분석, 관련 개념 비교, 시각화 등)이 떠오르면 문서에 임의로 추가하지 말고, 정리를 마친 뒤 사용자에게 제안한다.

## 스타일 규칙

- 제목은 `###`, 소제목은 `####` 수준으로 구성한다.
- 핵심 개념마다 적절한 이모티콘을 제목이나 목록 앞에 붙여 시각적으로 구분한다 (예: 🔹 개념, ⏱ 시간복잡도, 💡 팁, ⚠️ 주의, 📝 예제, ✅ 요약).
- 목록(`-`)과 표를 적극 활용해 정보를 스캔하기 쉽게 구성한다.
- 예제 코드는 코드블록(````언어명`)으로 표기한다.
- 흐름도, 트리/그래프 구조, 알고리즘 동작 과정 등 시각화가 이해에 도움이 되면 mermaid 코드블록(````mermaid`)을 사용한다.
- 핵심 요약은 문서 마지막에 `#### ✅ 요약` 섹션으로 짧게 정리한다.
- 불필요하게 장황한 설명은 피하고, 개념 → 예시 → 요약 순으로 간결하게 구성한다.
- 이미지/참고 링크가 있으면 인용문(`>`)이나 링크로 출처를 남긴다.
